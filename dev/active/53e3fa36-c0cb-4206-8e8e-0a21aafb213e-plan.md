## Proposed decomposition (for the breakdown node to fan out)

Right-sized tasks, grouped by subsystem. Gates: `cargo-ci` + `code-review` on every implementation task (per repo convention). Suffix shows dependencies.

**Wave 1 ‚Äî JSON contract (foundational; unblocks agent-side parsing)**
- T1 `issue show --json`: emit `short_id` at root (derive from `id[0:8]`); ensure `labels`/`dependencies`/`gates_required` always `[]`. Add per-lifecycle-state contract tests. *(R2)*
- T2 Unify list envelopes: `search --json` returns `{count, issues, ‚Ä¶}` (alias `results` for one release). *(R2, dep: none)*
- T3 `gate check --json`: add `verdict: pass|fail|pending` at root; human output prints `[PASSED]/[FAILED] <gate>` first line; add `--tail N`. *(R3)*

**Wave 2 ‚Äî gate workflow primitives**
- T4 `gate wait <ISSUE> <GATE> [--timeout N]`: block to terminal status, exit 0/1/2. *(S1; dep T3 for verdict)*
- T5 `gate status <ISSUE> [--json]`: per-issue gate table + `all_passed`. *(F6; dep T3)*
- T6 `gate pass` on FAIL: print first ~40 lines of checker stdout inline; `--quiet` suppresses. *(F2)*
- T7 `gate pass` already-passed short-circuit (`already_passed: true`); `gate pass-all <ISSUE>`. *(F7)*
- T8 `gate runs <ISSUE> [--gate K] [--json]`: run history newest-first. *(O6; dep T3)*
- T9 Document `gate pass` exit-code taxonomy in `--help`; add `verdict` to success `--json`. *(R1)*

**Wave 3 ‚Äî issue/dep/validate ergonomics**
- T10 `issue show --field <name>` / `--fields a,b` projection; multi-ID `issue show <id‚Ä¶> --json`. *(O1)*
- T11 Unified `gates` array in `issue show --json` (`{key,status,last_run_at}`, `pending` on add). *(O2; dep T1)*
- T12 `validate`: exclude `rejected`/`archived`/`done` from isolated-node check; add `--errors-only`. *(R4)*
- T13 `issue delete`: atomically remove inbound dep edges; `validate --fix` repairs dangling deps. *(S4)*
- T14 `issue search --label`; make QUERY optional when a filter is given. *(O3)*
- T15 `claim release <ISSUE_ID>` convenience + `--agent-id` on all `claim` subcommands. *(S3)*
- T16 `issue claim`: name `issue assign` in blocked-by-deps error; add `--assign-only`. *(S2)*
- T17 `graph deps --flat --json` flat array. *(R5)*

**Wave 4 ‚Äî discoverability & batch (lower ROI)**
- T18 Aliases: `dependency`‚Üí`dep`, `document`‚Üí`doc`, `issue list`‚Üí`query` redirect, `--add-label`‚Üí`--label`. *(O5)*
- T19 `issue delete --yes`/`-y`; document `JIT_ALLOW_DELETION` in `--help` + error. *(O4)*
- T20 `issue batch-create --from-json` (array + symbolic `depends_on`). *(S5)*
- T21 Gate-template `pass_context_lines` cap for test-runner gates. *(O7; may split to its own epic)*

**Dependency DAG**: T1‚Üí{T11}; T3‚Üí{T4,T5,T8}; T1‚ÜíT2 (independent ok). Waves are soft ordering; T1/T3 are the only hard upstreams. Roughly 21 tasks; Wave 1‚Äì2 (T1‚ÄìT9) deliver the bulk of the measured friction reduction.
