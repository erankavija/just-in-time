Overview · 2 / 16

The epic, at a glance

Residual friction that made the CLI hard for an agent to drive on the first try: guess-the-flag commands, truncated gate output, all-or-nothing validation.

  • 19 of 19 [hard] criteria (REQ-01 … REQ-19) delivered
  • 7 topological waves — each fully gated before the next began
  • Two whole-tree gates per child: cargo-ci (fmt + clippy + 2834 tests) and an AI code-review
  • Gates were inviolable: every review failure was reworked until it passed on merit
  • Net CLI surface: 2 commands retired, ~6 made first-guess-correct
First-guess CLI · 3 / 16

The first guess should just work

# before — the title was only reachable through a flag
jit issue create --title "Parse frontmatter"

# after — the title is the positional subject; the type is a flag
jit issue create "Parse frontmatter" --type task
#   -> Created issue: d3d666e1
REQ-01 / REQ-02. The flag form still works; the long nouns dependency and document stay as aliases of dep / doc.

REQ-04 / REQ-19. rdeps now defaults to immediate dependents; --depth 0 opts into the full transitive set.

Gates · 6 / 16

Pass a gate the way you reach for it

# before — the gate key was positional-only
jit gate pass abc123 ci

# after — positional OR flag, identical behaviour
jit gate pass abc123 --gate ci
REQ-03. The same flag lands on gate check; supplying both forms (or neither) is a machine-readable --json error, not a panic.
Gates · 7 / 16

Gate inspection becomes one surface

jit gate check abc123 ci                  # latest run (default, unchanged)
jit gate check abc123 --all               # full history, newest first
jit gate check abc123 --all --status failed   # filter by gate / outcome
jit gate check abc123 ci --stdout         # raw report text, verbatim
REQ-08. One verb — latest, history (--all/--limit), and flat output (--stdout/--stderr/--tail). Both check and check-all stay.
Gates · 8 / 16

The whole report, not the first 20 lines

  • Gate runs store and surface the complete stdout/stderr — no silent truncation (REQ-07)
  • The flat view is byte-exact: no injected trailing newline, no decoration
  • Gate prompts and gate output were reformatted for readability (REQ-09 / REQ-10)
$ jit gate check abc123 lint --stdout

Gates · 9 / 16

Edit a gate without hand-editing JSON

# before — open the registry file and hope the JSON stays valid
$EDITOR .jit/gates.json

# after — every mutable field is reachable from the CLI
jit gate update ci --title "Continuous Integration"
jit gate update ci --pass-context false --clear-prompt
REQ-16. Atomic write through the shared primitive; per-issue gate status is left untouched; --clear-* distinguishes "set" from "remove".
Canonical renames · 10 / 16

One home for gate configuration

# before — `jit registry` duplicated gate-registry config
jit registry add ci --title "CI" ...

# after — `jit gate` is the single surface
jit gate define ci --title "CI" --description "build + test"

$ jit registry list
error: unrecognized subcommand 'registry'
REQ-06. Clean rename, no shims; --example ported onto gate define; -s is now unambiguously --state.
Canonical renames · 11 / 16

Retiring a footgun

jit issue breakdown did flat, parent-centric breakdown: the wrong shape for a bracketed container, and it copied the parent's type label onto every child.

$ jit issue breakdown abc123
error: unrecognized subcommand 'breakdown'

# decompose with the bracket engine instead
jit apply plan epic-123      # scaffolds the plan -> breakdown -> impl spine
REQ-06. Command, dispatch, implementation, tests and the MCP tool all removed; the bracket engine (driven by the plan template in .jit/templates.toml) is untouched.
Validation · 12 / 16

Validate exactly what you mean

jit validate                      # whole repository (unchanged)
jit validate <full-issue-id>       # NEW — only this issue's rules
jit validate --scope epic-123     # a container's bracket subtree (gate checker)
  • REQ-12 — per-issue scoping; --explain shows which rules matched
  • REQ-13 — template anchors resolve a config-declared .jit/gates.json key (e.g. repo-validate), not only Rust presets
